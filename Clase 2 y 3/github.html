Como subir las cosas del repo local al repo de GIT?

Una vez que hicimos un trabajo en el repo local, lo que debemos hacer es:

  1. "ctrl+s" de guardado local
  2. Cerramos el browser
  3. Abrimos una terminal
  4. Usamos "git status" para ver el estado de las mdificaciones y archivos nuevos que hayaos creado.
  5. Usamos "git add ." para agregar esos archivos

Ahora, creamos una rama para subir ese documento al servidor, ya que queremos hacer una brunch(rama) por cada clase. Las ramas son una bifurcacion del "Main", con el que queremos agregar por ejemplo, ciertas funcionalidades a un sitio.En este caso, hacemos una rama de cada clase:

  1. Escribimos en la terminal el comando para generar una nueva rama: git checkout -b feature/(en este caso)clase3_html
  2. La terminal muestra el mensaje en el que notifica que ha creado una rama "Switched to a new branch "feature/clase3_html": significa que Git salto del Main a feature/clase3_html, rama en el que el contenido difiere de un lugar a otro.
  3. Ahora necesitamos subir esa rama nueva al repositorio de GitHub: ya tenemos el codigo local, tenemos localmente una rama nueva y necesitamos que esto suba al repositorio central donde tenemos todo, que es en este caso GitHub.
  4. Hacemos "git status"
  5. En verde nos aparecera una lista de los archivos que hayamos creado y que necesitemos subir
  6. Hacemos git comit -m"clase 3: Etiquetas basicas
  7. Hacemos Git status para ver el estado de las cosas
  8. Ponemos nuevamente git comit -m"clase 3: Etiquetas basicas
  9. Nos va a saltar la notficacion de que los cambios fueron gusrdados en la nueva rama
  10. Hacemos un nuevo git status: nos dira que ya localmente no hay mas nada para subir, como si el paquetito ya estubiera completo para poder enviarse.
  11. Hacemos git push pero como es una nueva rama debemos dar la orden de que suba al remoto con "x" nombre al origen.
  12. Nos va a saltar una nueva notificacion y deberemos copiar la parte que dice "git push --set-upstream origin+ (el nombre del repo local asociado). Se hace cuando creamos una rama local y queremos PUSHEAR al remoto.
  13. Hacemos entonces "git push --set-upstream origin/+ (nombre de nueva branch)"+ enter: esto sube los cambios al repositorio remoto de Git.
  14. Para verificar que esté la nueva brunch creada, ir a GitHub y chequear bajo el menu del Main.

Ahora necesitamos que la nueva rama(branch) forme parte del Main, para adicionarlos cambios que estuvimos trabajando:

  1. Hacemos en la terminal: git status
  2. Vemos si modificamos algo mas, hacemos git add .
  3. Luego informamos el cambio con git commit -m"(comentario)", en este caso por ejemplo voy a usar "cambio para mostrar pull request"
  4. y hacemos git push, NO estamos creando una nueva rama en este caso, sólo agregando cambios a una rama preexistente
  5. Nos va a saltar un listado de informacion de lo que se acaba de hacer, entre ellas informar el sitio el repositorio al que se sumaron los cambios en remoto.
  6. Hacemos ctrl+click derecho sobre direccion de git hub que aparece como "to: direccion http". Se abre browser.
  7. En git hub: vamos a hacer pull request: nos a aparece una ventana donde esta la rama con los cambios que queremos adicioar al main.
  8. Si queremos hacemos un comentario.
  9. Pull request + confirmamos el marge, que suma las modificaciones al Main.
  10. Ahi terminamos en el remoto

Puede ser que abramos de nuevo una rama en el visual(local). En la terminal deberemos:

  • git checkout main: abrimos l rama principal
  • Hacer git pull: trae los cambios hechos en remoto a local.
  • vamos a la rama en la queremos trabajar: git checkout +tab (permite buscar entre ramas)+enter

Si no nos abriera los archivos de las otras ramas para poder trabajar podemos traer todo lo que se esta haciendo en el main a la rama en la que estamos queriendo trabajar. El comando es git pull origin main: nos trae todo lo que esta en remoto a la rama en la que estamos trabajando
